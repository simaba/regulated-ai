# Changelog

All notable changes to this project will be documented in this file.

## [0.1.0] — Foundation release

### Added

- Regulated AI starter kit structure for governance, risk, release, incident response, and model-card artifacts.
- Practical adoption guide explaining what to customize in the first hour and first week.
- Generic sample release checklist and sample risk register.
- Release readiness report and deployment decision record templates.
- Governance artifact presence checks through GitHub Actions.
- Release checklist YAML validation workflow for starter and sample configuration files.
- README guidance clarifying that this is a starter kit, not legal advice, compliance certification, or a substitute for formal review.

### Notes

This repository is intended to be forked and customized. Public examples should remain generic and free of customer, employee, vendor, supplier, or internal project data.

### Next

- Add stronger schema validation for release and risk artifacts.
- Add more industry-specific examples while keeping them generic and public-safe.
- Add clearer mapping from starter-kit artifacts to `governance-playbook`, `release-governance`, and `release-checklist` workflows.
